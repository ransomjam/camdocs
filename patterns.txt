
# PATTERNS FOR HIERARCHICAL NUMBERING CORRECTIONS

## 1. PATTERNS DETECTING HIERARCHICAL RELATIONSHIPS IN HEADINGS

### Pattern A: Sequential Major/Minor Topics
(?i)^\s*(\d+(?:\.\d+)*)\s+([A-Z\s]{2,})(?:\s+|$)\n^\s*(\d+(?:\.\d+)*)\s+([A-Z\s]{2,})(?:\s+|$)

### Pattern B: Temporal/Categorical Relationships
(?i)(\d+(?:\.\d+)*)\s+(WEEK|MONTH|YEAR|QUARTER|TERM|SEMESTER)\s+([A-Z\d]+)(?:\s+|$)

# Followed by:
(\d+(?:\.\d+)*)\s+(DAY|SESSION|CLASS|PERIOD|LECTURE)\s+([A-Z\d]+)(?:\s+|$)

### Pattern C: General → Specific Patterns
(?i)^\s*(\d+(?:\.\d+)*)\s+([A-Z][^\.]{5,30})(?:\s+|$)\n^\s*(\d+(?:\.\d+)*)\s+([A-Z][^\.]{5,30})(?:\s+|$)

## 2. COMMON HIERARCHICAL PATTERNS

### Pattern D: Week/Day Pattern
(?i)(\d+\.\d+)\s+WEEK\s+([A-Z\d]+).*?\n(\d+\.\d+)\s+DAY\s+([A-Z\d]+)

### Pattern E: Unit/Lesson Pattern
(?i)(\d+\.\d+)\s+UNIT\s+([A-Z\d]+).*?\n(\d+\.\d+)\s+LESSON\s+([A-Z\d]+)

### Pattern F: Chapter/Section Pattern
(?i)(\d+\.\d+)\s+CHAPTER\s+([A-Z\d]+).*?\n(\d+\.\d+)\s+SECTION\s+([A-Z\d]+)

### Pattern G: Module/Topic Pattern
(?i)(\d+\.\d+)\s+MODULE\s+([A-Z\d]+).*?\n(\d+\.\d+)\s+TOPIC\s+([A-Z\d]+)

## 3. PATTERNS WITH ROMAN NUMERALS/LETTERS

### Pattern H: Lettered Hierarchies
(?i)(\d+\.\d+)\s+([A-Z])\s+.*?\n(\d+\.\d+)\s+(\d+)\s+.*?

### Pattern I: Roman Numeral Hierarchies
(?i)(\d+\.\d+)\s+(?:CHAPTER|PART|SECTION)\s+([IVXLCDM]+).*?\n(\d+\.\d+)\s+([A-Z\d]+)

## 4. CONTEXTUAL HIERARCHY DETECTION

### Pattern J: Short Title Followed by Specific Title
^(\d+\.\d+)\s+([A-Z]{2,15})\s*$\n^(\d+\.\d+)\s+([A-Z].{10,})

### Pattern K: Category/Subcategory Pattern
(?i)^(\d+\.\d+)\s+(TYPES|CATEGORIES|CLASSIFICATIONS|FORMS|MODELS).*?\n^(\d+\.\d+)\s+(.*?MODEL|.*?TYPE|.*?FORM)

## 5. NUMERICAL SEQUENCE PATTERNS

### Pattern L: Same Parent Number Sequence
^(\d+)\.(\d+)\s+.*$\n^\1\.(\d+)\s+.*$

### Pattern M: Consecutive Numbering with Semantic Clues
(\d+\.\d+)\s+.*?\b(GENERAL|OVERVIEW|INTRODUCTION|BASIC|FUNDAMENTAL|MAIN)\b.*?\n(\d+\.\d+)\s+.*?\b(SPECIFIC|DETAILED|ADVANCED|IMPLEMENTATION|APPLICATION|CASE)\b.*?

## 6. CONTENT-BASED HIERARCHY DETECTION

### Pattern N: Heading Similarity Detection
(\d+\.\d+)\s+.*?\b(COST\s+SHARING)\b.*?\n(\d+\.\d+)\s+.*?\b(COST\s+SHARING|FUNDING|FINANCE|TUITION)\b.*?

### Pattern O: Acronym/Full Form Pattern
(\d+\.\d+)\s+.*?\b([A-Z]{2,})\b\s+.*?\n(\d+\.\d+)\s+.*?\b(?:Definition|Meaning|Explanation|Overview)\b.*?

## 7. STRUCTURAL HIERARCHY PATTERNS

### Pattern P: List-like Structures
(\d+\.\d+)\s+.*?\n(\d+\.\d+)\s+(?:[0-9]+[\.\)]\s+.*?|[a-z]\)\s+.*?|•\s+.*?|\-\s+.*?)

### Pattern Q: Procedure/Steps Pattern
(\d+\.\d+)\s+.*?\b(PROCEDURE|STEPS|METHOD|PROCESS|APPROACH)\b.*?\n(\d+\.\d+)\s+(?:STEP|PHASE|STAGE)\s+\d+

## 8. CORRECTION PATTERNS

### Pattern R: Convert Flat to Hierarchical
(?i)^(\d+\.\d+)\s+(WEEK|UNIT|MODULE|CHAPTER|PART)\s+([A-Z\d]+)(?:\s+|$)\n^(\d+\.\d+)\s+(DAY|LESSON|TOPIC|SECTION|SESSION)\s+([A-Z\d]+)(?:\s+|$)

### Pattern S: Increment-Based Correction
^(\d+\.\d+)\s+(.*?)\n^(\1(?=\D)|\d+\.\d+)\s+(.*?)$

## 11. VALIDATION PATTERNS

### Pattern T: Validate Correct Hierarchy
(?i)^(\d+(?:\.\d+)*)\s+.*?\n^(\1\.\d+)\s+.*?$

### Pattern U: Detect Incorrect Flat Hierarchy
(?i)^(\d+\.\d+)\s+.*?\b(WEEK|UNIT|CHAPTER|MODULE)\b.*?\n^(\d+\.\d+)\s+.*?\b(DAY|LESSON|TOPIC|SESSION)\b.*?$

### Pattern V: Suggest Corrections
(?i)^(?P<parent_num>\d+\.\d+)\s+(?P<parent_title>.*?\b(?:WEEK|UNIT|CHAPTER)\b.*?)\n^(?P<child_num>\d+\.\d+)\s+(?P<child_title>.*?\b(?:DAY|LESSON|TOPIC)\b.*?)$

# Correction suggestion:
# Change: {child_num} {child_title}
# To: {parent_num}.1 {child_title}

# Correction function for APA reference formatting
def format_apa_references(text):
    """
    Fixes common APA reference formatting issues:
    1. Adds hanging indent
    2. Ensures consistent punctuation
    3. Italicizes journal/book titles
    4. Sorts alphabetically
    5. Adds missing information
    """
    references = text.strip().split('\n')
    formatted_refs = []
    
    for ref in references:
        # Fix capitalization in titles
        ref = re.sub(r'([A-Z][a-z]+)\s+([A-Z][a-z]+):', r'\1 \2:', ref)
        
        # Ensure period after year
        ref = re.sub(r'(\(\d{4}(?:/\d{4})?\))\s*([A-Z])', r'\1. \2', ref)
        
        # Fix journal formatting
        ref = re.sub(r'(\bJournal\s+of\s+[A-Z][a-z]+\s+[A-Z][a-z]+)', r'*\1*', ref)
        
        # Add hanging indent placeholder
        if len(ref) > 80:
            ref = ref[:80] + '\n    ' + ref[80:]
        
        formatted_refs.append(ref)
    
    # Sort alphabetically by first author/organization
    formatted_refs.sort(key=lambda x: re.sub(r'^([A-Z][A-Za-z]+).*', r'\1', x).lower())
    
    return '\n'.join(formatted_refs)

--- SHORT DOCUMENT DETECTION CRITERIA ---

A document is classified as "short" when:
1. Word count < 3000 words, OR
2. Estimated pages < 10 (2000 chars per page), OR
3. Line count < 500 lines, OR
4. Contains short document indicators (Assignment, Homework, etc.), OR
5. Heading/section count < 10

A document is NOT short if it contains long document indicators:
- "Dissertation", "Thesis", "PhD", "Master's Thesis"
- "Literature Review", "Abstract"

--- SHORT DOCUMENT PROCESSING ---

1. TOC REMOVAL:
   - Detect TOC header (TABLE OF CONTENTS, Contents)
   - Skip all TOC content lines (with dot leaders, page numbers)
   - End TOC detection on: 2 blank lines OR real heading

2. KEY POINT EMPHASIS:
   - Detect key point type from pattern match
   - Add emoji prefix based on category
   - Apply formatting: Bold (most), Italic (examples)

3. ASSIGNMENT HEADER:
   - Detect metadata fields (Student Name, Course, etc.)
   - Apply bold formatting

--- HELPER METHODS ---

- is_short_document(text) - Returns (is_short, reason)
- is_toc_header_line(text) - Check if line is TOC header
- is_toc_content_line(text) - Check if line is TOC entry
- remove_toc_from_lines(lines) - Remove TOC section
- get_key_point_type(text) - Returns (type, emoji) or (None, None)
- is_assignment_header_field(text) - Check if line is assignment field
- emphasize_key_point(text, type, emoji) - Apply emphasis formatting
- process_short_document(text) - Full short document processing

   - Tables (highest) -> Headings -> References -> Lists -> Definitions -> Others

2. Heuristic detection supplements pattern matching:
   - ALL CAPS short text (< 60 chars, <= 6 words) -> H1
   - Title Case short text (< 60 chars, 2-8 words) -> H2

4. Context-aware detection considers previous/next lines

5. Each line is processed exactly ONCE to prevent duplication

   and flagged with needs_page_break=True in the analysis result
